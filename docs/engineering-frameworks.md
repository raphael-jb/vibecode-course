# Engineering Frameworks (Reference)

## 1. Implementation Planning (`writing-plans`)
Before writing any code for a new feature or complex change:
1.  **Extract Spec:** Clarify visual and functional goals.
2.  **Task Breakdown:** Split into 2-5 minute tasks.
3.  **Explicit Verification:** Define how to prove it works.
4.  **Simplicity First:** Follow YAGNI.

## 2. Systematic Debugging (`systematic-debugging`)
1.  **Trace:** Find root cause via inspection.
2.  **Reproduce:** Create minimal reproduction.
3.  **Fix:** Apply minimal change to root cause.
4.  **Verify & Harden:** Prove fix and prevent recurrence.

## 3. Test-Driven Development (TDD)
- **Red:** Write failing test/visual state.
- **Green:** Minimal code to pass.
- **Refactor:** Clean up while staying green.

## 4. Visual Sync Verification (`visual-sync`)
1.  **Request Screenshot:** Proactively ask user.
2.  **Visual Diff:** Compare against Brand DNA.
3.  **Auto-Correct:** Propose "Premium Polish" if generic.
